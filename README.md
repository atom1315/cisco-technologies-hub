Cisco Technologies Hub

Методология по технологиям Cisco — от основ сетевой инфраструктуры до продвинутых решений автоматизации и безопасности.


О проекте
Этот репозиторий — структурированная база знаний по ключевым технологиям Cisco. Он создан для сетевых инженеров, студентов и всех, кто хочет систематизировать свои знания в области сетевых технологий. Здесь собраны описания протоколов, архитектурных решений и практические примеры конфигураций.

Маршрутизация (Routing)
Маршрутизация — фундамент любой сети. Cisco предлагает широкий набор протоколов для построения надёжных и масштабируемых сетей.
OSPF (Open Shortest Path First) — протокол маршрутизации на основе состояния канала. Используется в крупных корпоративных сетях благодаря быстрой сходимости и поддержке иерархической структуры через зоны (areas). OSPF работает внутри одной автономной системы и поддерживает аутентификацию между соседями.
EIGRP (Enhanced Interior Gateway Routing Protocol) — проприетарный протокол Cisco, сочетающий свойства дистанционно-векторных и link-state протоколов. Отличается быстрой сходимостью за счёт алгоритма DUAL и экономным использованием полосы пропускания, отправляя обновления только при изменениях в топологии.
BGP (Border Gateway Protocol) — основной протокол междоменной маршрутизации в Интернете. BGP управляет обменом маршрутной информацией между автономными системами и позволяет реализовывать сложные политики маршрутизации через атрибуты путей.

Коммутация (Switching)
VLAN (Virtual LAN) — технология логического разделения сети на изолированные широковещательные домены. Позволяет гибко управлять трафиком, повышать безопасность и упрощать администрирование без изменения физической топологии.
STP (Spanning Tree Protocol) — протокол предотвращения петель на канальном уровне. Cisco поддерживает несколько реализаций: классический STP (802.1D), Rapid STP (802.1w) с ускоренной сходимостью и Per-VLAN STP+ (PVST+), который создаёт отдельное дерево для каждого VLAN.
EtherChannel — технология агрегации каналов, объединяющая несколько физических линков в один логический. Поддерживает протоколы согласования LACP (стандарт IEEE) и PAgP (проприетарный Cisco), обеспечивая увеличение пропускной способности и отказоустойчивость.

Безопасность (Security)
ASA (Adaptive Security Appliance) — аппаратно-программный межсетевой экран Cisco, обеспечивающий фильтрацию трафика, VPN-подключения и инспекцию приложений. ASA использует концепцию уровней безопасности интерфейсов для контроля доступа между сегментами сети.
Cisco ISE (Identity Services Engine) — платформа управления доступом к сети на основе политик. ISE реализует аутентификацию по стандарту 802.1X, профилирование устройств и интеграцию с Active Directory для централизованного контроля.
ACL (Access Control Lists) — механизм фильтрации трафика на маршрутизаторах и коммутаторах. Стандартные ACL фильтруют по IP-адресу источника, расширенные — по адресам, портам и протоколам. ACL применяются как на входящем, так и на исходящем направлении интерфейса.

Беспроводные сети (Wireless)
Cisco WLC (Wireless LAN Controller) — контроллер для централизованного управления точками доступа. WLC обеспечивает единую точку настройки политик безопасности, роуминга и качества обслуживания. Архитектура CAPWAP используется для связи между контроллером и точками доступа.
Cisco DNA Center — платформа управления сетью на основе принципов intent-based networking. Позволяет автоматизировать развёртывание, мониторинг и устранение неполадок как проводных, так и беспроводных сегментов через единый интерфейс.

Автоматизация и программируемость
Автоматизация — ключевой вектор развития современных сетей. Ручное управление десятками и сотнями устройств больше не масштабируется. Cisco активно развивает программные интерфейсы и инструменты, позволяющие перейти от CLI-ориентированного администрирования к полноценному Infrastructure as Code.
Cisco IOS XE — современная модульная операционная система с поддержкой программных интерфейсов RESTCONF и NETCONF, а также модели данных YANG. Это позволяет описывать желаемое состояние устройства в структурированном виде (XML/JSON) и применять конфигурацию через API-вызовы вместо ручного ввода команд. IOS XE также поддерживает гостевые контейнеры (Guest Shell), в которых можно запускать Python-скрипты и Linux-утилиты прямо на маршрутизаторе.
Cisco NX-OS и ACI — операционная система для коммутаторов Nexus в дата-центрах. NX-OS предоставляет NX-API — REST-интерфейс для программного управления. Cisco ACI (Application Centric Infrastructure) выводит автоматизацию на уровень всей фабрики ЦОД, управляя сетевыми политиками через контроллер APIC с помощью декларативного подхода.
Cisco SD-WAN (Viptela) — решение для программно-определяемых глобальных сетей. Разделяет плоскость управления и данных, обеспечивая централизованное управление политиками, выбор оптимального пути и шифрование трафика между филиалами. Контроллер vManage предоставляет REST API для полной автоматизации жизненного цикла WAN-инфраструктуры.
Cisco Meraki — облачная платформа управления сетью. Meraki Dashboard API позволяет автоматизировать провижининг устройств, управление сетями и сбор аналитики через простые RESTful вызовы. Идеально подходит для распределённых организаций с сотнями филиалов.

DevOps и NetDevOps
Концепция NetDevOps переносит практики разработки ПО в мир сетевого администрирования. Цель — сделать сетевые изменения предсказуемыми, тестируемыми и воспроизводимыми.
Infrastructure as Code (IaC)
Ansible и Cisco — наиболее популярная связка для автоматизации. Коллекции cisco.ios, cisco.nxos, cisco.aci и cisco.meraki покрывают практически весь спектр оборудования. Ansible-плейбуки описывают желаемое состояние сети декларативно, а идемпотентность модулей гарантирует безопасное повторное применение. Типичный рабочий процесс включает генерацию конфигураций из Jinja2-шаблонов, валидацию и пуш на устройства.
Terraform и Cisco — провайдеры Terraform для ACI, Meraki и ISE позволяют управлять сетевой инфраструктурой наравне с облачными ресурсами. Terraform хранит состояние (state) и отслеживает дрифт конфигурации, что особенно ценно в гибридных средах, где сеть и облако управляются из единого пайплайна.
Python и библиотеки — Netmiko, Napalm и pyATS/Genie составляют основу Python-инструментария для работы с Cisco. Netmiko упрощает SSH-взаимодействие с устройствами. Napalm предоставляет унифицированный API для получения и сравнения конфигураций от вендоронезависимого интерфейса. pyATS — фреймворк Cisco для автоматизированного тестирования сетей, незаменимый в CI/CD-пайплайнах.
CI/CD для сетевой инфраструктуры
Современный подход к управлению сетью включает полноценный пайплайн непрерывной интеграции и доставки:

Git — хранение конфигураций и плейбуков в системе контроля версий обеспечивает аудит, откат и совместную работу команды
Линтинг и валидация — yamllint, ansible-lint и pyATS-проверки выполняются автоматически при каждом коммите
Тестирование в лабораторной среде — Cisco CML (Cisco Modeling Labs) позволяет развернуть виртуальные копии сетевого оборудования и прогнать изменения перед продакшеном
Автоматический деплой — после прохождения тестов конфигурации применяются на оборудование через Jenkins, GitLab CI или GitHub Actions
Мониторинг после деплоя — pyATS health checks и Genie diff автоматически проверяют, что изменения применены корректно

Контейнеризация и сетевые сервисы
Cisco NSO (Network Services Orchestrator) — платформа оркестрации мультивендорных сетей. NSO использует YANG-модели для абстракции устройств и транзакционный подход к применению изменений: либо конфигурация применяется на всех устройствах сервиса, либо происходит полный откат. Поддерживает интеграцию с Docker и Kubernetes для развёртывания сетевых сервисов как микросервисов.
Model-Driven Telemetry — замена традиционному SNMP-polling. Устройства Cisco самостоятельно стримят данные о своём состоянии (CPU, память, интерфейсы, BGP-соседи) через gRPC или NETCONF в реальном времени. Данные поступают в стек мониторинга — Telegraf, InfluxDB, Grafana (TIG) или Prometheus, что позволяет строить дашборды и алерты с минимальной задержкой.

Мониторинг и диагностика
NetFlow / Flexible NetFlow — технология анализа сетевого трафика, собирающая информацию о потоках данных. Позволяет выявлять аномалии, планировать ёмкость сети и расследовать инциденты безопасности. Flexible NetFlow расширяет возможности, позволяя настраивать произвольные ключевые поля для экспорта.
SNMP и Syslog — стандартные протоколы мониторинга. SNMP используется для сбора метрик с устройств (загрузка CPU, состояние интерфейсов), а Syslog — для централизованного сбора и анализа логов событий. В DevOps-среде логи направляются в ELK-стек (Elasticsearch, Logstash, Kibana) или Graylog для централизованного анализа.
Cisco ThousandEyes — платформа мониторинга сети и приложений от Cisco, обеспечивающая сквозную видимость от пользователя до облачного сервиса. Интегрируется с CI/CD-пайплайнами для проактивного обнаружения деградации производительности после сетевых изменений.
